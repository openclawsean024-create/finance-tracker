## 1. 問題陳述

### 1.1 目標使用者

| 族群 | 規模 | 痛點 |
|---|---|---|
| 中小企業主（10-50 人公司） | ~15 萬 | 會計記帳慢、月報表製作耗時、現金流看不到 |
| 工作室/接案者 | ~30 萬 | 多帳戶分散（公司戶/個人戶/PayPal/Stripe）、月底對帳痛苦 |
| 新創公司財務長 | ~5,000 | 沒有會計師也能快速產出投資人級的月報表 |

### 1.2 為什麼不做替代方案

- **Excel 自製表**：無自動化、容易公式錯誤、版本管理混亂
- **會計師事務所**：月費 5,000-20,000 NT$，對微型企業太貴
- **商用會計軟體（Xero/QuickBooks）**：年費 12,000-30,000 NT$，介面複雜
- **我們的解法**：純前端 + CSV 匯入 + Recharts 視覺化，零月費、30 秒產出月報表

---

## 2. 解決方案

### 2.1 核心價值主張

> 「30 秒看完這個月賺多少、花多少、還有多少現金可用。」

### 2.2 使用者流程

1. 首次設定：公司基本資料 + 帳戶清單（銀行/現金/數位錢包）
2. 從銀行/平台匯出 CSV 交易明細，匯入系統
3. 系統自動分類（薪資/租金/營收/進貨等），可手動調整
4. 儀表板即時顯示：本月營收、本月支出、應收應付、現金水位
5. 月底一鍵匯出 PDF 月報表給會計師/投資人

## 3. 功能清單

### 3.1 MVP（必做）

- [ ] 帳戶管理（銀行/現金/PayPal/Stripe 等 6 種）
- [ ] CSV 匯入交易明細（支援台新/國泰/中信三家銀行格式）
- [ ] 交易自動分類（12 種預設類別 + 自訂）
- [ ] 應收/應付管理（到期提醒）
- [ ] 月報表 PDF 匯出
- [ ] 儀表板 5 大 KPI（營收/支出/現金/應收/應付）

### 3.2 v2（加值）

- [ ] 多公司支援（集團/工作室主）
- [ ] 銀行 API 自動串接（需銀行授權）
- [ ] 預算設定 + 超支警示
- [ ] 發票 OCR 自動建檔

### 3.3 明確不做

- 線上付款（不做金流）
- 稅務申報（交給會計師/國稅局軟體）
- 多幣別即時匯率（先支援 NTD/USD 兩種）
- 多語系介面

## 4. 技術棧

| 層 | 選擇 | 理由 |
|---|---|---|
| 前端 | React 18 + Vite + TypeScript | 與既有架構一致 |
| 圖表 | Recharts | 純 React、生態完整、輕量 |
| CSV 解析 | Papa Parse | 業界標準、支援 bigint |
| PDF 匯出 | jsPDF + html2canvas | 前端純 PDF 產生、無後端依賴 |
| 資料持久化 | localStorage（明文）+ IndexedDB（CSV 暫存） |
| 部署 | Vercel |

## 6. 風險與決策

| 風險 | 等級 | 緩解 |
|---|---|---|
| 銀行 CSV 格式變動 | 🔴 高 | 抽象化 CSV 解析層；單元測試每家銀行格式；提供手動對應 UI fallback |
| 數字錯誤導致法律/稅務問題 | 🔴 高 | 明確聲明「不替代會計師」；數字加 tooltip 顯示計算來源 |
| localStorage 容量限制（大量歷史交易） | 🟠 中 | 月底自動打包、只留 12 個月熱資料在 localStorage |
| 個資/財務資料外洩（裝置失竊） | 🟠 中 | v2 加 PIN 碼鎖定；明確隱私頁聲明 |
